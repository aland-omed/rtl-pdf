# Roadmap

## 0.1 - Native text MVP

- [x] Unicode BiDi visual runs
- [x] Arabic-script OpenType shaping
- [x] Mixed RTL/LTR numbers and codes
- [x] Mirrored punctuation
- [x] Grapheme-safe wrapping
- [x] Pagination and document metadata
- [x] Logical `/ActualText`
- [x] ESM and CommonJS package validation

## 0.2 - Document primitives

- [x] Ordered and unordered lists
- [ ] RTL tables with repeating headers
- [ ] Images, links, and annotations
- [ ] Headers, footers, and page numbers
- [ ] Multiple font fallbacks by Unicode coverage
- [ ] Hyphenation and language-aware line breaking

## 0.3 - Ecosystem adapters

- [ ] PDFKit adapter API
- [ ] pdf-lib adapter
- [ ] React components
- [ ] jsPDF adapter
- [ ] pdfmake adapter

## 0.4 - Accessibility

- [ ] Tagged document structure
- [ ] Reading-order validation
- [ ] PDF/UA validation fixtures
- [ ] Alternative text and language spans
- [ ] Screen-reader testing

## 1.0 - Stable engine

- [ ] Unicode BiDi conformance suite
- [ ] Arabic, Kurdish, Persian, Urdu, Hebrew, and Pashto golden files
- [ ] Browser and serverless bundles
- [ ] Stable layout and plugin APIs
- [ ] Performance benchmarks and compatibility guarantees

## Community projects

- Professional invoice, report, contract, certificate, and receipt templates
- Public browser playground
- Font compatibility catalogue
- Examples for Next.js, Express, Cloudflare, AWS Lambda, and Bun
- Upstream fixes and integrations for major JavaScript PDF libraries
